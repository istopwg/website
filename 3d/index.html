3D Printing

The Printer Working Group is defining an extension to the Internet Printing Protocol and PWG Semantic Model for 3D Printing. The extension leverages our existing network protocol and Job Ticket formats and one or more existing high-level 3D file formats to describe the objects to print. We have a prototype-ready draft available now and welcome participation from all interested parties.

Summary of Work

We started our work in "Birds of a Feather" (BOF) sessions at our face-to-face meetings and on the 3d-printing mailing list we host. A white paper was produced outlining our approach, and that lead to the addition of the IPP 3D Printing Extensions to the Internet Printing Protocol workgroup charter. While IPP and the PWG Semantic Model can be easily adapted to 3D printing, adapting the existing 3D file formats is proving to be more of a challenge.

3D printers can take advantage of the 50+ years of traditional 2D printing to offer high-level, networked, and cloud-aware printing services to client devices running any operating system. The Internet Printing Protocol provides a flexible and secure network protocol for queuing jobs, monitoring the state of jobs and printers, and managing jobs and printers. When combined with a high-level 3D file format, the proper job ticket attributes (i.e., print options), and one or more service discovery or directory protocols (Bonjour/DNS-SD, LDAP, etc.) a client device will be able to successfully print 3D objects to any printer using "generic" software.

Adopted 3D File Formats

3D Manufacturing File Format (3MF): 3MF offers a slightly more compact XML format than AMF with physical dimensions, named materials, and shared vertices. 3MF is supported by major CAD and slicer software and is a required file format for the IPP 3D Printing Extensions. More Info

PDF: PDF 1.7 (ISO 32000-1) includes 3D support using the Universal 3D format ("U3D", ECMA-363) and PDF 2.0 (ISO 32000-2) will add support for the Product Representation Compact format ("PRC", ISO 14739-1:2014) format. Both U3D and PRC are binary file formats with named materials. PRC also includes manufacturing tolerance metadata. PDF is a recommended file format for the IPP 3D Printing Extensions. More Info on ECMA-363 More Info on ISO 32000-1

Other 3D File Formats (Considered but Not Adopted)

Additive Manufacturing File Format (AMF): AMF is an ISO standard XML format that supports physical dimensions, named materials, and shared vertices. It is generally considered to be the replacement for STL and is supported by some 3D software. The lack of a freely-available version of the standard has limited its use in 3D printing. More Info

Collada (DAE): Collada is an XML format for 3D assets that is widely used in gaming. While it does offer shared vertices, it does not provide a way to name materials or define physical units. More Info

Stereo Lithography File Format (STL): STL is the current defacto-standard file format with both plain text and binary encodings. While it is the most widely used and supported file format for 3D printing, it lacks support for physical dimensions, named materials, or shared vertices. More Info
